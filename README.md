一个轻量、开箱即用的 Python 工具，专为**批量整理图片命名**而设计。  
支持自定义基础文件名、首张编号、递增步长、统一后缀，以及“覆盖原文件 / 输出到新文件夹”两种输出模式。

---

## 🌟 项目亮点

- 多格式支持：`jpg` / `jpeg` / `png` / `gif` / `bmp`
- 自定义命名：可输入并确认基础文件名
- 编号可控：支持首张后缀编号（如 `01`）+ 递增数（`1/2/3`）
- 后缀可选：保持原后缀，或统一成指定后缀
- 输出灵活：覆盖当前文件名，或保存到新文件夹
- 自动创建新文件夹：名称使用“基础文件名”（自动处理非法字符）
- 顺序稳定：按文件名排序后执行重命名
- 冲突保护：遇到重名自动跳过并寻找下一个可用序号
- 新手友好：全流程交互式输入，按提示即可完成

- 设计/开发资源目录规范化

## 🧠 命名示例

假设基础文件名输入为 `product`，首张编号输入为 `01`，递增数选择 `2`，后缀统一为 `.png`，则输出结果类似：

- `product_01.png`
- `product_03.png`
- `product_05.png`

若输出模式选择“保存到新文件夹”，会在当前目录下创建：

- `./product/`

## 🚀 快速开始

### 1) 环境要求

- Python 3.8 或更高版本

### 2) 运行方式

#### 方式 A：交互式运行（推荐）

```bash
python rename_images.py
```

脚本会依次提示：

1. 图片目录路径
2. 基础文件名（并确认）
3. 是否统一后缀
4. 首张图片后缀编号（如 `01`）
5. 后续递增数（`1/2/3`）
6. 输出模式（覆盖原文件 / 保存到新文件夹）

#### 方式 B：命令行参数运行

```bash
python rename_images.py <文件夹路径> <补零位数> <起始编号> <是否递归> <基础文件名> <后缀> <递增数> <是否保存到新文件夹>
```

参数说明：

- `<文件夹路径>`：目标图片目录
- `<补零位数>`：例如 `2` 会生成 `01`
- `<起始编号>`：首张编号，例如 `1`、`10`
- `<是否递归>`：`0` 表示否，`1` 表示是
- `<基础文件名>`：例如 `product`
- `<后缀>`：如 `png`、`.jpg`，传 `0/none/keep/original` 表示保留原后缀
- `<递增数>`：只支持 `1/2/3`
- `<是否保存到新文件夹>`：`0` 覆盖原文件，`1` 保存到新文件夹

示例：

```bash
python rename_images.py ./images 2 1 0 product png 2 1
```

含义：处理 `./images` 当前目录，基础名 `product`，后缀统一为 `.png`，编号从 `01` 开始按 `2` 递增，结果保存到新建文件夹 `./images/product/`。

## ⚠️ 使用建议

- 若选择“覆盖当前文件名”，建议先备份原始图片
- 建议先在少量样本目录试跑参数
- Windows 下可直接粘贴带引号路径，脚本会自动处理
- 选择“保存到新文件夹”时，原图会保留，新图会复制到新目录
